## Orchestration engine migration (COMPLETE 2026-07-06)

All seven workflow skills — `code`, `plan`, `prd`, `research`, `agent`, `sca`, `jsa` — now run on the shared `orchestration` engine, each a bespoke `BasePlaybook` subclass with a ~5-line delegate `orchestrate.py`. `rez` is the lone exception: a placeholder awaiting a dedicated build. The migration replaced every per-skill `python-statemachine` FSM, the `--state` argv, and the `/tmp/<skill>-<sid>.json` state model with a durable `run_id`-keyed SQLite checkpointer. Dead code was removed repo-wide (legacy driver path −642 lines, jsa/sca legacy modules, the `docs/agents/skills/examples/` tree), docs were rewritten engine-first, `scaffold-skill.py` regenerated for the engine, and the two legacy-model CI validators (`check_skill_structure`, `check_compliance`) updated. An adversarial review found and fixed 13 defects (sca contract-vs-prompt type mismatches, a `code` verify↔learning livelock, plan escalation gaps, a research counter-reset bug, a jsa verify-doc overclaim). Engine suite 286 green, driver 107 green. Also done this arc: tiktoken made the canonical token counter, `SYSTEM.md` pruned to ~934 tok with all reference paths extracted into the AGENTS.md index chain.

## sca skill — tool layer

The sca security/correctness items about the pre-migration `orchestrate.py` FSM — crafted `--state` blob / tampered `/tmp/sca-<sid>.json` state, `_gate_cleared`/`_cleared_gates` type confusion, augment-cap disk tampering, and the `restore_state` disk trust-anchor (old Findings A/B/C, the per-gate-boolean structural hardening recommendation) — are **superseded by the engine migration**. The engine owns state in a durable `run_id`-keyed checkpointer (no `--state` argv, no `/tmp` state file), obviating that entire tamper surface; sca preserves every human gate, the augment-loop cap (=3), and the single-shot verification through the engine's `GATE_STATES`/`route_user`. Full history: the `wing_decisions/sca-skill` drawer. The old output_dir basename-collision → cross-repo contamination risk is **fixed** (domain code now derives the dir from `{basename}-{shorthash(absolute target_path)}`).

- [ ] **Fresh audit: engine gate/checkpoint hardening.** The adversarial review of the migrated playbooks covered contracts + loop honesty, but did not re-run the old sca type-confusion / state-tamper threat class against the *engine's* gate and checkpointer handling (a filesystem-capable subagent could in principle touch `.penny/orchestration.db`). Scope a dedicated pass if that threat model matters.
- [ ] **Exit-code classification for trufflehog / njsscan / eslint-security.** These 3 tool extensions still use the looser generic `if (err.stdout)` truthy heuristic (can misclassify a crash-that-emits-stdout as a findings run); osv-scanner and trivy got proper `classify*Exit` functions. Fix: document each tool's real exit-code convention and apply classify functions. Low real-world likelihood until the tools are actually provisioned.
- [ ] **codeql full scan-path (DB create + analyze) unverified.** codeql 2.25.6's version-check is real-verified; the heavy two-step DB build + analyze path is opt-in/off-by-default (behind `confirm_opt_in`) and intentionally not exercised in tests. Verify it and its SARIF `runs[].results[]` parser against a real DB build when codeql scanning is first used.
- [ ] **[Deferred v2] Tool-install automation for the 8 optional/required sca tools.** Real curl-based install + SHA256 verification for osv-scanner, gitleaks, trufflehog, trivy, retire.js, njsscan, eslint plugins, codeql. Deferred to avoid fabricating checksums; `provisioning.py` TOFU + `tool_manifest.py` version pins already provide the safety property at real provisioning time. Acceptance: each script pins a version, verifies a real upstream SHA256, respects license tiers (copyleft = invoke-only), and no-ops if the tool is present.
- [ ] **[Deferred] NodeGoat benchmark ground-truth is an honest empty template.** `compute_benchmark_metrics()` is fully unit-tested on synthetic fixtures; activating the benchmark needs a human to clone the real OWASP NodeGoat repo, verify entries (each with a `_verified_by` provenance note), populate `tests/nodegoat/ground-truth.json`, and point `$SCA_NODEGOAT_PATH` at the clone.
- [ ] **P10 sandbox defense-in-depth (`sandbox.py`, still live).** Beyond the required minimum (verified-zero Linux capabilities + active seccomp/AppArmor): (a) the container runs as UID 0 internally — add `--user 65534:65534` as a further layer against a hypothetical kernel/runc escape; (b) the base image `alpine:3.19` is tag-pinned, not digest-pinned — digest-pin it for supply-chain reproducibility when this moves toward production provisioning.
