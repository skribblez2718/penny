# SEMANTIC UNDERSTANDING

Interpret the query's true meaning before proceeding.

## Instructions

Execute the following analysis on the user's query:

1. **Interpret Semantic Meaning**
   - Look beyond the literal words to understand the underlying intent
   - What is the user actually trying to accomplish?
   - What problem are they trying to solve?

2. **Identify Task Domain**
   - Classify the task domain: technical / personal / creative / professional / recreational / hybrid
   - Note any hybrid aspects if multiple domains apply
   - Consider how domain affects approach

3. **Determine Appropriate Approach**
   - What tool, method, or approach offers the best first-attempt success?
   - Consider available skills and agents
   - Match complexity of approach to complexity of task

4. **Contextual Awareness**
   - Remember: Today's date is the current system date, NOT training data cutoff
   - Consider any recent context from previous interactions
   - Note any time-sensitive aspects of the request

## Output Requirements

After processing this step, clearly articulate:
- The interpreted intent behind the query
- The identified task domain with confidence level
- The recommended initial approach
- Any contextual factors that influence the approach
