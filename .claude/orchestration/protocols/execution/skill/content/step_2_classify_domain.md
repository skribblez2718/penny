# Classify Task Domain

Classify the task into the appropriate domain to guide cognitive processing.

## Available Domains

| Domain | Description | Examples |
|--------|-------------|----------|
| **Technical** | Software, systems, engineering | API development, debugging, architecture |
| **Personal** | Life decisions, health, goals | Career advice, habit building, relationships |
| **Creative** | Art, writing, content creation | Stories, designs, creative content |
| **Professional** | Business, career, workplace | Strategy, management, presentations |
| **Recreational** | Fun, games, entertainment | Game recommendations, hobby projects |
| **Hybrid** | Multi-domain requiring mixed approach | Technical + Personal, Creative + Professional |

## Classification Criteria

Consider:
1. **Primary objective:** What is the main goal?
2. **Required expertise:** What knowledge areas apply?
3. **Success metrics:** How will success be measured?
4. **Stakeholders:** Who benefits from the outcome?

## Output Requirements

Classify and output:
1. Primary domain selection
2. Secondary domain (if hybrid)
3. Justification for classification

**Format your response as:**
```
PRIMARY DOMAIN: {technical|personal|creative|professional|recreational}
SECONDARY DOMAIN: {domain or "none"}
JUSTIFICATION: {brief explanation of classification}
```
