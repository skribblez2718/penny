Bounded revise loops & honest exit

  • critiquing_plan ⇄ planning and critiquing_report ⇄ synthesizing (deep only) and validating ⇄ synthesizing (all modes) each loop on NEEDS_REVISION/FAIL, bounded by max_iterations. Budget spent → the "exhausted" edge proceeds forward with a warning recorded (never a faked pass).
  • validating (vera) is the evidence-based citation-grounding gate in every mode — the report is written only after claims are grounded (or the ground-loop budget is spent).

Fan-out & escalation

  • researching (echo) fans out one branch per planned sub-query (dynamic_branches, bounded by max_fan_width); quick mode with no sub-queries stays a single echo agent.
  • Any working phase escalates to unknown on UNCERTAIN; unknown → awaiting_clarification → planning resumes after the user answers.
  • Abort: every working phase can abort → error (arrows omitted for legibility).
