Validated against orchestration/playbooks/plan.py (PlanMachine). Decomposes a goal into a reviewable plan, then a task breakdown. Boxes are states dispatching cognitive-primitive agents; MemPalace is the blackboard.

Critique routing & honest exit

  • critiquing (carren, evidence-gated) has five routes: APPROVE → taskifying; NEEDS_REVISION → exploring (need more context, while explore_rounds < 2) or planning (fixable in the plan, once explore_rounds ≥ 2); budget spent → taskifying with met=False; BLOCKED (categorically-unsafe plan) → complete halted with met=False — a safety stop, never a faked pass.

Gate & escalation

  • verify_gate (HITL): the user confirms → critiquing, or revises → planning. Only entered when planning flags the plan high-stakes.
  • Any working phase escalates to unknown on UNCERTAIN; unknown → awaiting_clarification → scoping re-scopes after the user answers.
  • Abort: every working phase can abort → error (arrows omitted for legibility).
