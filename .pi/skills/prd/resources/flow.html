prd skill — execution flow

Validated against orchestration/playbooks/prd.py (PrdMachine). Turns a free-form brief into a formal spec + a structured IDEAL_STATE the code skill consumes. Boxes are states dispatching cognitive-primitive agents; MemPalace is the blackboard.
Smallest FSM, two working states: generating (synthia) does clarification-questions, full synthesis, and revision; validating (vera) is the evidence-gated schema + quality + traceability check. Clarification is not a gate — generating raises questions through the escalation subsystem and resumes on the user's answer.

Loops & honest exit

  • generating → generating (self-loop, synthesize): a clarification pass that yields zero new requirements proceeds straight to full synthesis rather than re-asking.
  • validating → generating (revise): issues found and within max_iterations. Budget spent → complete with met=False — never a faked pass.
  • done_predicate: PASS requires the PRD valid and the IDEAL_STATE valid (evidence-gated). Reaching complete via exhaustion is honest failure, flagged met=False.

Escalation (clarification lives here, not in a gate)

  • generating → unknown on needs_clarification (open questions for the user); validating → unknown on a stall (the same issues persist). unknown → awaiting_clarification → generating resumes with the answer.
  • Abort: both working phases can abort → error (arrows omitted for legibility).
