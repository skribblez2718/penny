Bounded loops & honest exit (never a faked pass)

  • learning → implementing (retry): bounded by max_iterations. Budget spent with a gap remaining → complete with met=False and the unresolved gaps reported.
  • learning ⇄ verifying (final battery): after learning reports no gap, one last full verify battery runs. Bounded by FINAL_VERIFY_CAP = 3 — a persistent learn(no-gap)/verify(fail) disagreement completes honestly (met=False) instead of spinning.
  • done_predicate: met = (learn_gap is False) AND (verify_passed). Reaching complete does not imply success — only that predicate does.

Escalation subsystem (dashed)

  • Any working phase (exploring, analyzing, checking_criteria, planning, implementing, verifying, learning) escalates to unknown on UNCERTAIN confidence; learning also escalates on a stall / repeated strategy (progress_check).
  • unknown → awaiting_clarification → exploring: the run pauses for the user, then resumes at exploring with the clarification.

Other true-to-code notes

  • Two HITL gates: criteria_gate (refine / accept / skip) and plan_gate (approve / refine / deny — deny is a terminal error). No code is written before plan_gate approval.
  • Autonomy pre-check on implementing (AUTONOMY_STATES) is dormant unless PENNY_AUTONOMY_GATE is set.
  • Abort: every working phase can abort → error; those arrows are omitted for legibility (only plan_gate deny is drawn).
