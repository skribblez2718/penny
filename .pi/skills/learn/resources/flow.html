Bounded loops & honest exit (never a faked pass)

  • authoring ⟲ / assessing ⟲: self-loop once per lesson (author_next/assess_next, lesson_count from the design SUMMARY); progress is tracked in the checkpointer, so crash-resume continues at the same lesson.
  • verifying ⇄ fixing and critiquing → fixing share ctx.iteration against max_iterations (default 3). Fixes ALWAYS re-verify against the whole corpus — cross-file forks are invisible to single-file checks.
  • Honest exhaustion: budget spent → complete with met=False and unresolved_violations reported (verify_exhausted / critique_exhausted) — never a fabricated pass. Stalled identical violations escalate via progress_check instead of spinning.

Escalation subsystem (dashed)

  • Every working state (scoping, ingesting, designing, authoring, assessing, synthesizing, verifying, fixing, critiquing) escalates to unknown on UNCERTAIN / needs_clarification; verifying also escalates on stalled violations.
  • unknown → awaiting_clarification → designing: the run pauses for the user, then resumes at designing with lesson progress preserved.

Other true-to-code notes

  • Ingest topology is model-emitted (Bitter-Lesson compliance): scoping (echo) emits ingest_branches; a caller constraints.ingest_branches skips scoping entirely (start_ingest). The 3-focus default (content / conventions / assessment) survives only as the tagged LOAN learn_default_ingest_topology — ablated with no caller topology, the run fails loud.
  • One HITL gate: charter_gate (approve / refine / deny — deny is a terminal error). Nothing is authored before approval; conventions locked here cannot drift later.
  • Abort: every non-final state can abort → error; those arrows are omitted for legibility (only charter_gate deny is drawn).
